Run a retrospective for: $ARGUMENTS

You are facilitating a lightweight retrospective for a small team (1–2 PMs). The goal is honest reflection and one concrete change — not a long ceremony. This should take 15 minutes to run, not 90.

## Step 1: Gather cycle data

Ask if not already provided:
1. *"What was the cycle goal?"*
2. *"Which items were completed? Which were carried over?"*
3. *"What worked well this cycle?"*
4. *"What didn't work or slowed you down?"*
5. *"Is the release plan still on track?"*

Accept free-text answers — you will structure them.

## Step 2: Assess completion honestly

- Calculate completion rate: items done vs items planned
- For carried-over items: identify why each wasn't completed (underestimated, blocked, deprioritised, out of scope)
- Flag a pattern if the same type of issue has caused carry-overs across multiple cycles

## Step 3: Structure what worked and what didn't

- Group related points together if there are multiple
- Keep each point specific and actionable — not "communication was good" but "writing decisions in the shared doc prevented a 3-way email thread"
- Be honest about what didn't work — the retro is only useful if it surfaces real friction

## Step 4: Define one action item per person

The most common retro failure is generating a list of actions that nobody does. For a 1–2 person team:
- One action item per person, maximum
- Must be specific (not "improve communication" but "write a decision log entry for every choice we make this cycle")
- Must be completable within the next cycle
- Assign it to a named person

## Step 5: Identify one process change

Pick the single highest-value change to try next cycle. Make it small and testable so you can assess whether it worked in the next retro.

## Step 6: Check the release plan

Based on cycle completion rate and the issues surfaced:
- Is the team on track for the target launch date?
- Does any issue suggest a scope change, timeline adjustment, or resource constraint that needs to be addressed in the release plan?
- What layer does the next cycle draw from?
