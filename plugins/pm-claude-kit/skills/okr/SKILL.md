Draft OKRs for: $ARGUMENTS

Use the template in [okr-template.md](okr-template.md) as the structure.

Follow this process:
1. Write 1-3 Objectives that are:
   - Qualitative and inspirational — describes the destination, not the path
   - Ambitious but achievable within the time frame
   - Clearly owned by a person or team
2. For each Objective, write 2-4 Key Results that are:
   - Quantitative and measurable — includes a specific number or percentage
   - Outcome-focused, not output-focused (avoid "launch X" — prefer "achieve Y% of users doing Z")
   - Graded on a 0.0–1.0 scale (0.7 = success, 1.0 = stretch)
3. For each KR, note:
   - Current baseline (if known or estimable)
   - Data source / how it will be measured
   - Owner
4. Flag any KRs that are activity-based (bad) vs. outcome-based (good).
5. Include a brief "Confidence Check" — would hitting these KRs confirm the Objective was truly achieved?

Output formatted OKRs ready to drop into a planning doc or presentation.
